# 水培农业管理系统 - 演示流程方案

## 演示概述

- **演示角色**：ADMIN（管理员）
- **演示账号**：`admin` / `admin123`
- **故事线**：模拟水培农场日常运营管理的完整工作流
- **建议时长**：约 25 分钟

## 演示前准备

### 1. 启动基础设施

```bash
docker compose up -d
```

### 2. 执行数据库迁移

```bash
docker compose exec -T mysql mysql -uroot -proot hydroponic < packages/backend/migrations/0001_init.up.sql
docker compose exec -T mysql mysql -uroot -proot hydroponic < packages/backend/migrations/0002_seed_auth.up.sql
docker compose exec -T mysql mysql -uroot -proot hydroponic < packages/backend/migrations/0003_seed_metrics.up.sql
docker compose exec -T mysql mysql -uroot -proot hydroponic < packages/backend/migrations/0004_notification_channels.up.sql
docker compose exec -T mysql mysql -uroot -proot hydroponic < packages/backend/migrations/0005_seed_devices.up.sql
docker compose exec -T mysql mysql -uroot -proot hydroponic < packages/backend/migrations/0006_seed_control_rules.up.sql
docker compose exec -T mysql mysql -uroot -proot hydroponic < packages/backend/migrations/0007_add_alerts_timestamps.up.sql
```

### 3. 启动后端

```bash
cd packages/backend && go run cmd/api/main.go
```

### 4. 启动前端

```bash
cd packages/frontend && npm install && npm run dev
```

### 5. 写入模拟遥测数据（可选但推荐）

由于没有真实 IoT 设备，演示前可调用 `POST /api/telemetry` 接口写入几条模拟传感器数据，使遥测图表和告警列表有内容可展示。写入后规则引擎会自动评估条件并生成告警记录。

## 演示流程

### 第一阶段：系统概览

#### 步骤 1 — 登录

| 项目 | 内容 |
|------|------|
| **页面** | `/login` |
| **操作** | 输入 `admin` / `admin123`，点击登录 |
| **展示要点** | JWT 认证流程，登录成功后自动跳转首页 |

#### 步骤 2 — 仪表盘

| 项目 | 内容 |
|------|------|
| **页面** | `/` 仪表盘 |
| **操作** | 展示首页全貌 |
| **展示要点** | 8 项聚合数据：设备在线/总数统计、告警统计（未处理/严重/今日）、温室环境概览卡片（均温/均湿）、近期控制指令时间线、设备类型分布饼图 |

> **讲解词**：作为系统首页，仪表盘让管理员一眼看到整个农场的运行状态——有多少设备在线、有多少告警需要处理、每个温室的温湿度概况。

---

### 第二阶段：资源管理

#### 步骤 3 — 温室管理

| 项目 | 内容 |
|------|------|
| **页面** | `/greenhouses` |
| **操作** | 查看 2 个预置温室（1号温室-叶菜类、2号温室-草莓） |
| **展示要点** | 温室 CRUD，展示种子数据中的温室信息（名称、位置、面积、种植作物） |

> **讲解词**：系统以"温室 → 设备组 → 设备"三层结构组织硬件资源。每个温室代表一个独立的种植区域。

#### 步骤 4 — 设备组管理

| 项目 | 内容 |
|------|------|
| **页面** | `/device-groups` |
| **操作** | 查看 6 个预置设备组 |
| **展示要点** | 每个温室下按功能分为 3 组：营养液监控组、环境监测组、执行设备组。展示设备组的设备绑定关系 |

#### 步骤 5 — 设备列表

| 项目 | 内容 |
|------|------|
| **页面** | `/devices` |
| **操作** | 查看 19 台预置设备列表 |
| **展示要点** | 筛选功能（按类型 SENSOR/ACTUATOR、在线状态、温室、分组）、批量操作模式（批量启用/禁用/删除） |

#### 步骤 6 — 设备详情

| 项目 | 内容 |
|------|------|
| **页面** | `/devices/13`（环境温度传感器） |
| **操作** | 点击设备进入详情页，切换到"数据面板"标签 |
| **展示要点** | 设备基础信息 + 数据面板：遥测摘要（时间范围选择器 + 指标图表）、在线率统计、关联告警事件列表。展示单设备维度的数据聚合分析 |

> **讲解词**：管理员可以在设备列表进行批量管理，也可以下钻到单设备查看详细的运行数据面板。

---

### 第三阶段：数据监控

#### 步骤 7 — 实时遥测

| 项目 | 内容 |
|------|------|
| **页面** | `/telemetry/realtime` |
| **操作** | 选择设备，查看最新传感器读数 |
| **展示要点** | 实时数据面板，展示各设备最新指标值（温度、湿度、pH、EC、CO2、光照），含计量单位 |

#### 步骤 8 — 历史数据

| 项目 | 内容 |
|------|------|
| **页面** | `/telemetry/history` |
| **操作** | 选择设备 + 指标类型 + 时间范围，查看历史数据 |
| **展示要点** | 分页历史数据查询，支持多维度筛选，展示从 MySQL 读取的时序数据记录 |

> **讲解词**：系统通过 MQTT 协议实时采集设备遥测数据，支持最新数据快照和历史数据回溯查询。数据双写至 MySQL 和 InfluxDB 时序数据库。

---

### 第四阶段：智能控制

#### 步骤 9 — 自动化规则

| 项目 | 内容 |
|------|------|
| **页面** | `/controls/rules` |
| **操作** | 查看 22 条预置自动化规则 |
| **展示要点** | 每条规则包含：触发条件（指标 + 比较运算符 + 阈值）、执行动作（指令类型 + 参数）、目标设备。规则有启用/禁用开关。展示**滞回区间**设计（高温开启 → 低温关闭，避免阈值边界抖动） |

#### 步骤 10 — 新建规则

| 项目 | 内容 |
|------|------|
| **页面** | `/controls/rules` |
| **操作** | 点击新建，演示规则创建对话框 |
| **展示要点** | 选择目标设备 → 设置触发条件（如 TEMP > 30°C） → 配置执行动作（如 FAN = ON）。展示规则创建的完整交互 |

#### 步骤 11 — 手动控制

| 项目 | 内容 |
|------|------|
| **页面** | `/controls/commands` |
| **操作** | 选择执行器设备（如排风扇），手动下发控制指令 |
| **展示要点** | 指令下发对话框、指令历史列表、指令状态流转（PENDING → SENT → EXECUTED / FAILED）。展示批量指令下发功能 |

> **讲解词**：系统支持两种控制模式——手动下发指令适合人工干预场景，自动化规则适合无人值守。规则引擎在每次遥测数据入库时自动评估条件并触发对应动作。22 条预置规则覆盖了温湿度、pH、EC、CO2、光照六大环境指标的自动调控，采用滞回区间设计防止阈值附近反复触发。

---

### 第五阶段：告警处理

#### 步骤 12 — 告警列表

| 项目 | 内容 |
|------|------|
| **页面** | `/alerts` |
| **操作** | 查看告警列表，使用筛选条件 |
| **展示要点** | 按告警类型（THRESHOLD 阈值告警 / OFFLINE 离线告警 / DEVICE_ERROR 设备故障）、级别（INFO / WARN / CRITICAL）、状态（OPEN / ACK / CLOSED）筛选。告警列表通过 JOIN 展示关联设备名称 |

#### 步骤 13 — 告警确认与关闭

| 项目 | 内容 |
|------|------|
| **页面** | `/alerts` |
| **操作** | 确认（ACK）一条告警，再将其关闭（CLOSED） |
| **展示要点** | 告警生命周期：OPEN（待处理）→ ACK（已知晓）→ CLOSED（已解决）。展示告警统计面板（未处理数/已确认数/已关闭数） |

> **讲解词**：当遥测数据触发规则阈值或设备离线时，系统自动生成告警。运维人员确认表示已知晓，处理完毕后关闭，形成完整的告警生命周期闭环。

#### 步骤 14 — 用户管理

| 项目 | 内容 |
|------|------|

| 项目 | 内容 |

## 演示注意事项

1. **遥测数据**：种子数据仅包含设备元数据和规则定义，不包含实际遥测数据。如需展示监控图表和告警功能，请提前通过 API 写入模拟数据。
2. **告警数据**：写入遥测数据后，规则引擎会自动评估并生成告警记录。
3. **通知分发**：当前版本告警生成后不会自动推送通知（通知调度尚未与告警生成联动），通知渠道页面可单独演示配置和测试功能。
4. **SSE 实时推送**：SSE 端点已注册但尚未实现完整的 `text/event-stream` 推送，实时数据页面通过轮询方式获取最新数据。
