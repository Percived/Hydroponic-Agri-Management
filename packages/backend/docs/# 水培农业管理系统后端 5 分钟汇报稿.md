# 水培农业管理系统后端 5 分钟汇报稿

各位老师好，我汇报的项目是“水培农业管理系统后端”。项目目标是把温室设备、传感器、控制指令和告警联成一条可追踪的闭环，支持水培场景的日常运维与自动化控制。

首先介绍架构与技术栈。系统采用 Go + Gin + GORM 构建服务端，MySQL 用于保存设备、用户、规则等主数据，InfluxDB 用于存储高频时序遥测数据，MQTT 作为设备通信通道，用于下发控制指令。整体流程是“设备上报 → 后端接入处理 → 数据查询与自动控制”。

目前系统具备的核心能力主要分为六类：

第一类是用户与权限。系统支持 JWT 登录和 RBAC 角色控制，分管理员、操作员、只读用户，确保不同角色访问不同接口。

第二类是设备与温室管理。可以管理温室、设备与设备分组，为后续采集和控制提供清晰的组织结构。

第三类是遥测数据采集与查询。设备可以定时上报温度、湿度、PH 等指标，系统支持查询最新值、历史曲线以及统计结果。

第四类是控制与规则。用户可以手动下发控制命令，例如开关风机；也可以设置自动规则，例如“温度超过阈值自动开风机”。

第五类是告警管理。当规则触发时系统会生成告警记录，支持告警列表查询、确认与关闭，便于运维跟踪。

第六类是系统概览与审计。提供全局统计看板与审计日志查询，帮助快速了解系统运行状态与操作历史。

系统的核心业务闭环是：设备定时上报数据，系统写入数据库并进行规则评估；若规则满足条件，自动生成控制指令并通过 MQTT 下发到设备，同时生成告警记录。这让系统实现了“数据驱动的自动控制”，减少人工干预，提升管理效率。

当前阶段的不足主要有两点：一是自动化测试仍待补齐，二是部分高级能力（如模板应用、告警订阅）还处于基础版本。下一步计划是完善测试体系、增强规则与告警可靠性，并进一步明确设备端协议与指令规范。

总体来说，该系统已经完成了水培场景 MVP 所需的关键链路，从设备接入、数据管理到自动控制和告警闭环都已打通，具备可演示和可扩展的基础能力。谢谢老师。
