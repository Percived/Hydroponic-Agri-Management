气候联动：触发来源如何指定？

目标：固定“单一采集通道”的温度值作为触发依据，避免跨温室与多探头互相干扰

方案 A（推荐）：显式指定 温室 + 设备 + 指标 + 采集通道
温室 *
选择温室（下拉）
设备 *
选择设备（下拉，按温室过滤）
指标 *
TEMP / HUM / CO2…（下拉）
采集通道 *
选择通道（按设备+指标过滤）
触发规则：只消费该“采集通道”的遥测值；其他温室/其他探头的 TEMP 不参与评估

显式、可解释

用户在 UI 上完整指定来源，审计与排障更直观；后端可严格校验归属关系。

方案 B：只指定“采集通道”（设备/指标由通道反推）
温室 *
选择温室（下拉）
采集通道 *
选择通道（按温室过滤）
UI 只选通道；设备/指标字段只读展示（由通道归属决定）

更省步骤

选择更少，但用户对“为什么触发/用哪个点位”可能不够直观（需要展示解释信息）。
